# 🐸 뇽죵이 설명서 — 슬래시 커맨드 가이드

> 채팅창에 **슬래시(`/`) + 커맨드**를 입력하면 뇽죵이가 자동으로 워크플로우를 실행합니다.
> 트리거 단어("분석", "자율" 등)만 입력해도 동일하게 동작합니다.
> 까먹었을 때: `?` 또는 `도움` 입력하면 전체 목록이 표시됩니다.

---

## 📋 커맨드 목록

### 🔧 개발 작업

| 커맨드 | 언제 쓰나? | 뇽죵이가 하는 일 |
|--------|-----------|-----------------|
| `/수정` | 코드 고칠 때 | 코드 수정 → 빌드(`tsc`) → 테스트 → 품질 검사 → 결과 보고 |
| `/디자인` | UI 만들 때 | Stitch로 디자인 생성 → HTML 추출 → 코드 반영 → 빌드 |
| `/기능` | 기능 하나 추가할 때 | 목표 분석 → 전문가 자문 → 코딩 → 테스트 → 보고 |

### 📊 분석 & 전략

| 커맨드 | 언제 쓰나? | 뇽죵이가 하는 일 |
|--------|-----------|-----------------|
| `/분석` | 복잡한 기능 구현할 때 | 목표 분석 → 모델 추천 → 메모리 조회 → 전문가 자문 → 코딩 |
| `/조사` | 기술/시장 조사할 때 | Tavily + Context7 + 웹 검색 → 분석 보고서 작성 |
| `/자문` | 빠른 전문가 의견 필요할 때 | 페르소나 자문 단독 실행 → 의견 정리 |
| `/발상` | 킬러 피처 뽑을 때 | 경쟁사 리뷰 수집 → JTBD 핑퐁 → 10x 필터 → 킬러 피처 도출 |

### 🚀 프로젝트 관리

| 커맨드 | 언제 쓰나? | 뇽죵이가 하는 일 |
|--------|-----------|-----------------|
| `/자율` | **새 프로젝트** 처음부터 끝까지 | PRD → Stitch 디자인 → 코딩 → 완료 보고 (기존 프로젝트 ❌) |
| `/규칙` | 뇽죵이가 삼천포 빠졌을 때 | 상태 확인 → 목표 재분석 → 품질 검사 → 워크플로우 복귀 |
| `/셋업` | 프로젝트 시작할 때 | `PROJECT_CONTEXT.md` + `PITFALLS.md` 자동 생성 |

### 💾 메모리 & 세션

| 커맨드 | 언제 쓰나? | 뇽죵이가 하는 일 |
|--------|-----------|-----------------|
| `/이어서` | 이전 대화 이어갈 때 | 메모리 검색 → 핸드오프 노트 읽기 → 맥락 복구 |
| `/저장` | 지금 상태 저장할 때 | Obsidian에 현재 진행 상태 즉시 저장 |
| `/메모리` | 과거 기록 찾을 때 | 메모리 검색 → 결과 보고 |

## 🎯 상황별 추천 가이드

```
"이 버그 좀 고쳐줘"          → /수정
"새 화면 만들어줘"           → /디자인
"로그인 기능 추가해줘"       → /기능
"결제 시스템 설계해줘"       → /분석
"이 기술 좋은지 알아봐줘"    → /조사
"경쟁사 대비 뭐가 부족해?"   → /발상
"새 앱 처음부터 만들어줘"    → /자율
"어제 하던 거 이어서"        → /이어서
"지금 저장해둬"              → /저장
```

## 📌 완료 시 보고 포맷 (뇽죵이가 매번 제공)

```
📌 다음 작업 추천 모델: [모델명]
   모드: [Fast | Planning]
   진행도: [Phase] — 분석 | 설계 | 구현 | 검증 | 배포 | 완료
   컨텍스트: [🟢 양호 | 🟡 잘림 | 🔴 새 세션 권장]
   추천 커맨드: [다음에 쓸 커맨드]
   → 이유: [왜 이 모델/커맨드인지]
   → 다음 작업: [구체적 할 일]
```

### 사용 가능한 모델 목록
| 모델 | 용도 |
|------|------|
| `Gemini 3.1 Pro (High)` | 복잡한 설계/아키텍처/디버깅 |
| `Gemini 3.1 Pro (Low)` | 중간 난이도 구현/리팩터링 |
| `Gemini 3 Flash` | 빠른 단순 작업/질문 답변 |
| `Claude Sonnet 4.6 (Thinking)` | 논리적 추론/복잡한 버그 분석 |
| `Claude Opus 4.6 (Thinking)` | 최고 난이도 설계/전략 수립 |
| `GPT-OSS 120B (Medium)` | 범용 코딩/문서 작성 |

> 💡 **팁**: 커맨드 없이 그냥 말해도 됩니다. "이거 고쳐줘", "분석해줘" 같은 트리거 단어만 있으면 뇽죵이가 알아서 워크플로우를 실행합니다.
